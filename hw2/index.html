Overview

In this homework, I built a geometric modeling and mesh processing program capable of evaluating, shading, modifying, and upsampling 3D models.

In the first section, I implemented de Casteljau's algorithm to evaluate parametric Bezier curves and surfaces, rendering smooth curves from control points. In the second section, I used the half-edge data structure to manipulate triangle meshes. First I computed the area-weighted vertex normals to allow for smooth Phong shading across the surface of the meshes, then I implemented local edge flips and edge splits. This involved updating twins, nexts, vertices, edges, and faces to ensure the mesh remained manifold and didn't crash the program. Finally, I implemented Loop subdivision, which upsamples coarse triangle meshes to be high-resolution, smooth approximations by mathematically weighting and repositioning vertices Overall, I was able to transform simple topological inputs into smooth, complex 3D geometry.

The most interesting thing I learned was how deeply intertwined a mesh's visual behavior is with its underlying mathematical and topological structure. Working with the half-edge data structure taught me how important pointer management is. For instance, correctly using the isNew boolean flags to differentiate between original split-line edges and newly created cross-edges was crucial to preventing incorrect edge-flipping during upsampling.

Additionally, my debugging journey during the Loop subdivision phase was fascinating. I learned that uneven vertex degrees—such as the unequal corners on a standard triangulated cube—cause asymmetric deformations during upsampling because the weighted average calculations rely directly on the vertex degree. Mathematically balancing the mesh topology by pre-splitting edges to equalize vertex degrees before subdividing was the most satisfying part.

Section I: Bezier Curves and Surfaces

Part 1: Bezier curves with 1D de Casteljau subdivision

De Casteljau's algorithm recursively applies linear interpolation to the points, taking n-1 steps to eventually land on one interpolated resulting point. First, I created an empty std::vector to store the intermediate points. Then, I iterated through the given control points, applying the 1D de Casteljau linear interpolation formula n-1 times to calculate and append each intermediate point for the next subdivision level.


Part 2: Bezier surfaces with separable 1D de Casteljau

The de Casteljau algorithm can be applied multiple times to evaluate a 3D surface with essentially the same logic as the Vector2D version. This time, the evaluation step still applied the same formula to each level via a for loop, and then I implemented evaluate1D to recursively evaluate through all the points until only one remained, rather than doing just one subdivision. Finally, I implemented a grid evaluation function that uses double parameters u and v so that it can evaluate each row using u, and the resulting column using v.


Section II: Triangle Meshes and Half-Edge Data Structure

Part 3: Area-weighted vertex normals

To implement area-weighted vertex normals, I calculated the sum of the area-weighted normals of all faces connected to a specific vertex. First I initialized a 3D VectorSum and set up my constant iterator HalfedgeCIter h = half-edge() to traverse over the half-edges pointing away from the current vertex. I used a do-while loop to iterate through each adjacent triangle face and get the 3D coordinates of each vertex of the current triangle using half-edge pointers (h->vertex()->position, h->next()->vertex()->position, and h->next()->next()->vertex()->position). Then I was able to create two outward pointing vectors and cross them to get the normal vector which has a magnitude proportional to the triangle's area. I added the cross product to my normalSum and advanced to the next face using h = h->twin()->next(). Finally I returned the normalized normalSum.unit().


Part 4: Edge flip

To implement edge flipping, I first added a check to return the edge if it was on a boundary loop using the condition if e0->isBoundary(). Then I collected every half-edge, vertex, edge, and face in the local neighborhood (two adjacent triangles in a diamond shape) of my edge and assigned them to variables so I could use them later. Then I reassigned all five pointers for the inner-half edges using setNeighbors() and also reassigned halfedge() points for the vertices, edges, and faces in the diamond. While implementing this, I initially forgot to reassign the halfedge() pointer for one of the faces, so flipping an edge back and forth caused a segmentation fault. I used check_for() to see which objects were pointing at the face and caught the mistake.

Part 6: Loop subdivision for mesh upsampling

To implement Loop subdivision for mesh upsampling, I first iterated through all vertices in the original mesh and calculated their new positions in Vertex::newPosition using the Loop subdivision rule. I also iterated through the original edges and calculated where the new midpoint vertices would go. To subdivide, I used a for loop to iterate through the original edges, calling splitEdge(), marking the new vertex with isNew=true, and transferring the previously calculated e->newPosition into the new vertex. Then I iterated over all the edges and flipped only the new cross-edges that connected a new and old vertex using an XOR logic check. Finally, I went through all the vertices and updated their 3D position using the values stored in newPosition, marking isNew=false for further subdividing.

During implementation, my mesh was getting mangled and subdividing completely wrong. I realized this was because I was flipping the wrong edges; I hadn't flagged the two half-edges of the original line to isNew=false and the two cross-edges as isNew=true inside my splitEdge function. Once I fixed these flags, the flip loop accurately targeted the cross-edges and the subdivision worked as expected.

I observed that when Loop subdivision is applied, sharp corners and edges act like they are "melting" and become entirely smooth. This smoothing effect can be reduced by pre-splitting edges directly around the sharp corner, which increases the local vertex density and keeps the weighted average calculations much closer to the corner's original shape.

Furthermore, when subdividing the standard cube, it was noticeably asymmetric. This happens because the 8 corners of the initial cube, which are divided by single diagonal edges on each face, have different vertex degrees. As a result, they get pulled by different amounts during the position update calculations and deform the mesh. I was able to get a perfectly symmetrical result by pre-processing the cube: I split the diagonal edges on all 6 faces to create an 'X', evening out the vertex degrees of every corner so they were all exactly 6. Because the topology was balanced, it subdivided into a perfectly symmetrical sphere.
